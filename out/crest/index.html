🔐 3. Encryption Ciphers💡 Mnemonic — key/block sizes
DES = 56-bit key, 64-bit block — "5 before 6, 6 before 4" (D=4th letter → 64 block).
3DES = 3×DES → key 168, block stays 64.
AES = block always 128; keys go 128 → 192 → 256 (+64 each).
RC4 = Rivest Cipher 4 = stream (R for "river — flows bit by bit").
RSA = primes + asymmetric (named after authors).

CipherKey SizeBlock SizeType
DES💡 DES — Data Encryption Standard
56-bit key, 64-bit block. "5 before 6, 6 before 4".
Born 1977, broken by brute force — the weakest standard cipher you'll see on the exam.
Worst variant: DES-ECB (no chaining → patterns leak). 		56 bits64 bitsSymmetric block
3DES💡 3DES — Triple DES
Apply DES three times → effective key 112 (2-key) or 168 (3-key).
Block stays 64 (it's still DES underneath).
Replaced by AES because it's slow. 		112/168 bits64 bitsSymmetric block (DES ×3)
AES (Rijndael)💡 AES — Advanced Encryption Standard
Block always 128. Keys: 128 → 192 → 256 (+64 each step).
Original name Rijndael (Dutch authors) — selected by NIST to replace 3DES.
Common modes: CBC (TLS), GCM (auth+enc, used in TLS 1.3, WPA3-GCMP). 		128, 192, 256128 bitsSymmetric block
RC4💡 RC4 — Rivest Cipher 4
Stream cipher (no block size — encrypts bit-by-bit, "flows like a river").
Variable key 40–2048 bits.
Used in old WEP and early SSL/TLS — now considered broken. 		40–2048 bitsStreamSymmetric stream
RSA💡 RSA — Rivest, Shamir, Adleman
Asymmetric: public/private keypair built on the hardness of factoring large primes.
Min cert key size on exam: 2048 bits.
Uses: key exchange, digital signatures, PGP, X.509 certs. 		2048+ bits—Asymmetric
SYSKEY💡 SYSKEY — Windows SAM protection
128-bit symmetric key that encrypts the local SAM hive (where NTLM hashes live).
Mode 1 (default) stores it in registry — offline attacks can extract it with the SAM & SYSTEM hives (e.g. secretsdump). 		128 bits—Symmetric
  • DES = least secure / weakest • DES ECB = weakest form of DES
  • ECB mode = discloses structure of clear-text💡 ECB — Electronic CodeBook
    Each block encrypted independently → identical plaintext blocks produce identical ciphertext blocks.
    Famous "ECB penguin" image leak. Never use in real systems.
  • CFB mode = uses block cipher as stream cipher💡 CFB — Cipher FeedBack
    Turns a block cipher into a stream cipher by XOR-ing keystream from previous ciphertext.
    "Feedback = output of last step becomes input of next".
  • CBC mode = vulnerable to padding oracle attacks; IV is first block of ciphertext💡 CBC — Cipher Block Chaining
    Each block XOR'd with previous ciphertext → needs an IV (first "previous block").
    Weakness: padding oracle (POODLE, Lucky13) when server leaks pad-validity.
  • GCM (Galois/Counter Mode) = commonly used for AES on UDP streaming💡 GCM — Galois/Counter Mode
    Combines CTR mode (encryption) with Galois field math (auth tag) → AEAD.
    Authenticated encryption: gives both confidentiality AND integrity in one pass.
    Default in TLS 1.3, WPA3 (GCMP), and IPsec for UDP/streaming.
  • Rijndael = replacement for 3DES (became AES)
  • Diffie-Hellman = key exchange, vulnerable to MITM
  • RSA = asymmetric, uses prime numbers
  • Min SSL cert key size = 2048 bits
  • Cisco Type 7 = weak, reversible (Vigenère algorithm)
  • SHA-256 = cannot be used for reversible encryption
  • S/MIME = encrypt email • PGP = secure email (RSA, keyring)
  • SSHv2 = confidentiality + integrity, NOT vulnerable to MITM
  • RADIUS = only encrypts password • TACACS = encrypts entire packet
  • DTLS = Datagram Transport Layer Security (for UDP)
  • Block cipher = encrypts blocks • Stream cipher = encrypts 1 bit at a time
  • TLS downgrade attacks commonly target TLS 1.0

🌍 7. IP Addresses & Subnets

IPv4
32 bits, 4 bytes, 4 octets
Layer 3 (Network) in OSI
IPv6
128 bits, 16 bytes
Loopback: ::1
First 64 bits = network; last 64 = host (Interface ID)
Uses NDP instead of ARP
HSRPv2 supports IPv6 (v1 does not)

Private IP Ranges (RFC 1918)💡 Mnemonic — RFC 1918 ("Ten, Twelve, Sixteen")
10.0.0.0/8 — biggest block, 1 octet locked → /8 (Class A).
172.16.0.0/12 — odd one. Remember "16 → /12": starts at 172.16, ends at 172.31.
192.168.0.0/16 — home routers, 2 octets locked → /16 (Class C-ish).

The CIDRs (8 → 12 → 16) climb by +4 as the range shrinks.

RangeCIDR
10.0.0.0/8
172.16.0.0–172.31.255.255/12

Subnet Quick Reference💡 Mnemonic — subnets without a calculator
Usable hosts = 2^(32-CIDR) − 2. Just double-and-add-2 going DOWN:
/30 = 2 → /29 = 6 → /28 = 14 → /27 = 30 → /26 = 62 → /25 = 126 → /24 = 254
Pattern: each step doubles the previous +2.

Mask octet (the last non-255 octet):
128, 192, 224, 240, 248, 252, 254, 255
Each = 256 − block_size. Block sizes halve: 128→64→32→16→8→4→2→1.

Classful shortcut: /8=A, /16=B, /24=C.

CIDRSubnet MaskUsable Hosts

Hosts = 2^24 − 2 = 16,777,214.
First octet is network, last 3 are host. Example: 10.0.0.0/8. 		255.0.0.016,777,214 (Class A)

ICMP Types

TTL Default Values → OS Identification💡 Mnemonic — Powers of 2, round DOWN
OS defaults are powers of 2: 64, 128, 255.
When you receive a TTL, round UP to the nearest of these → that's the originator's OS.
Example: TTL 56 → was 64 (Linux), 8 hops away.
Example: TTL 123 → was 128 (Windows), 5 hops away.

Memory: Linux = Low (64) • Windows = "Win-128" • Cisco maxes the field (255).

📶 9. WiFi / 802.11 Protocols💡 Mnemonic — WiFi security evolution
WEP → WPA → WPA2 → WPA3. Each fixed the prior one's break.
Cipher progression: RC4 (WEP, broken) → RC4+TKIP (WPA bandage) → AES-CCMP (WPA2) → AES-GCMP (WPA3).
Standard: 802.11i = WPA2 amendment.
Auth: 802.1X = port-based, uses RADIUS + EAP.

Frequencies: 2.4 GHz, 5 GHz, 6 GHz, 60 GHz💡 Mnemonic — 802.11 letters & bands
802.11b/g/n = 2.4 GHz (longer range, more congestion).
802.11a/n/ac = 5 GHz (faster, shorter range).
802.11ax (Wi-Fi 6/6E) = 2.4 + 5 + 6 GHz.
802.11ad/ay = 60 GHz (WiGig, line-of-sight).
Memory: vowels (a/ax) go HIGH frequency; later letters add bands.
ProtocolStandardCipherKeyNotes
WEP💡 WEP — Wired Equivalent Privacy
RC4 with a tiny 24-bit IV → IV collisions after ~5,000 packets.
"128-bit WEP" is actually 104 secret + 24 IV.
Broken by FMS (Fluhrer/Mantin/Shamir) attack — needs ~100k–1M packets.
Memory: "WEP = Weakness Equals Pwnable". 		802.11a/bRC464/128-bit staticIV=24 bits, very weak
TKIP💡 TKIP — Temporal Key Integrity Protocol
Bandage on RC4 between WEP and WPA2. Adds:
• Per-packet key mixing
• 64-bit MIC (Michael) for integrity
• Larger 48-bit IV
Still uses RC4 → deprecated in WPA3. Memory: "Temporary fix". 		802.11iRC4Dynamic64-bit MIC, legacy compatible
WPA/WPA2💡 WPA2 — the modern baseline
Standard: 802.11i. Cipher: AES-CCMP (replaces TKIP).
Two modes:
• Personal (PSK) — passphrase, brute-forceable via 4-way handshake capture.
• Enterprise — 802.1X + RADIUS + EAP.
WPA3 upgrades to AES-GCMP + SAE (replaces PSK). 		802.11iAES + TKIP—CCMP, GCMP, PSK support
  • WEP = Wired Equivalent Privacy • WPA = WiFi Protected Access
  • WPA2 standard: 802.11i • WPA-Personal: PSK, susceptible to brute-force
  • TKIP = Temporal Key Integrity Protocol (interim WEP replacement)
  • 128-bit WEP = only 104 secret bits (24-bit IV)
  • WEP weakness: weak Initialization Vectors • Incorrectly implemented: RC4
  • FMS attack: ~100,000–1,000,000 packets needed
  • SSID: max 32 bytes, case sensitive, broadcast in clear text
  • Hidden SSID: still in Beacon Frames; revealed by Association Request
  • Monitor Mode = passive wireless frame capture
  • Deauth attack: recover hidden ESSID + capture WPA/WPA2 handshake
  • 802.1X: RADIUS with EAP extensions
  • EAP-RSA = NOT a valid EAP method
  • PEAP = Microsoft • LEAP = Cisco💡 EAP family — who made what
    EAP = Extensible Authentication Protocol (framework, not auth itself).
    PEAP = Protected EAP — Microsoft (uses server PKI cert, tunnels MSCHAPv2).
    LEAP = Lightweight EAP — Cisco (uses TKIP + dynamic keys, broken).
    EAP-TLS = mutual cert auth (most secure).
    EAP-TTLS = Tunneled TLS (server cert only).
    Memory: P=Protected=Pretty PKI, L=Lightweight=Lame & Cisco.
    EAP-RSA is NOT a valid EAP method (exam trap).
  • PEAP uses server-side PKI; LEAP uses TKIP + dynamic keys
  • Tools: Fiddler, WiFite, airmon-ng, Fern Wifi

chmod Permissions💡 Mnemonic — "R-W-X = 4-2-1"
Add the bits to get the digit:
r = 4 • w = 2 • x = 1

Three digits = owner / group / other.
Worked example — chmod 455 file.txt:
• 4 = 4+0+0 → r-- (owner)
• 5 = 4+0+1 → r-x (group)
• 5 = 4+0+1 → r-x (other)
Result: -r--r-xr-x

umask = bits to subtract from default (777 dir / 666 file).

  • find / -perm -4000 → locate SUID binaries (priv-esc hunting ground)
  • umask 0027 → new files = 640, new dirs = 750

📖 14. Abbreviations💡 Mnemonic — group them, don't memorize them

Cipher modes (all end in B/M/P): ECB, CBC, CFB, GCM, CCMP, GCMP.
Trick: C-modes for chaining, G-modes for Galois (auth).

Markup-Language family (all "ML"): XML, HTML, SAML = Security Assertion ML. AJAX = "Async JS + XML".

Email: SMTP sends, POP3/IMAP receive, S/MIME + PGP encrypt.

Routing: RIP (hop count) < IGRP < OSPF (link-state) < BGP (inter-AS).
First-Hop Redundancy (FHRP): HSRP (Cisco), VRRP (standard), GLBP (load-balance).

Windows IDs: SID = full id, RID = last chunk (RID 500 = Admin).

DNS records: A/AAAA (IP), CNAME (alias), MX (mail), SOA (zone authority), NS (nameserver), PTR (reverse).

"Asymmetric" / PKI: RSA, PGP, PKI, X.509 all live together.

Risk Management💡 Mnemonic — "ARAT" 4 treatments
Avoid • Reduce • Accept • Transfer

Primary objective: reduce risk to an acceptable level — NOT eliminate it entirely.

Inherent risk = baked-in risk of an activity before any controls.
Residual risk = what's left after controls are applied.
Formula: Risk = Likelihood × Impact.

TreatmentMeaningExample
AvoidStop doing the risky activityDon't run the legacy service at all
ReduceApply controls to lower likelihood/impactPatch, segment, harden
AcceptAcknowledge and live with itLow-impact risk, sign-off by risk owner
TransferShift to a third partyCyber insurance, outsourcing
  • Inherent risk = the risk implicitly associated with the activity itself
  • Residual risk = remaining risk after controls (always > 0)
  • Risk register = living document tracking all identified risks + treatments
  • Executive Summary → business audience (impact) • Technical findings → IT/InfoSec

Authentication Factors & Biometrics💡 Mnemonic — biometric accuracy
FAR = False Accept Rate — impostor let in (security risk).
FRR = False Reject Rate — legit user blocked (usability risk).
CER = Crossover Error Rate = point where FAR = FRR.

Lower CER = better system.

The 3 MFA factor types: something you Know / Have / Are (= K-H-A).

FactorTypeExamples
Something you knowKnowledgePassword, PIN, security question
Something you havePossessionSmart card, TOTP token, FIDO2 key
Something you areInherence (biometric)Fingerprint, iris, face, voice
Somewhere you areLocationGPS, IP geolocation, network zone
  • MFA = at least 2 different factor types (two passwords ≠ MFA)
  • Iris scan = most accurate biometric • Fingerprint = most common
  • FAR vs FRR tradeoff: tightening one loosens the other; CER is the balance point

RAID Levels💡 Mnemonic — RAID at a glance
0 = Zero redundancy (stripe only) — speed, no safety.
1 = m1rror — pure copy, halves capacity.
5 = stripe + 1 parity — survives 1 disk loss (≥3 disks).
6 = stripe + 2 parity — survives 2 disk losses (≥4 disks).
10 (1+0) = mirror, then stripe — speed + safety, ≥4 disks.

Memory: Even = no parity (0/10), 5 = 1 parity, 6 = 2 parity.

RAIDTechniqueMin DisksFault Tolerance
0Striping2None — any disk fails = data lost
1Mirroring2Survives 1 disk loss; 50% capacity
5Stripe + distributed parity3Survives 1 disk loss
6Stripe + double parity4Survives 2 disk losses
10 (1+0)Mirror, then stripe4Survives multiple losses (not in same mirror)
  • RAID-1 = "disk mirroring" (classic exam phrasing for fault tolerance)
  • RAID is for availability, NOT backup — ransomware/deletion replicates instantly
